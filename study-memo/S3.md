# 概要

Amazon S3（Simple Storage Service）は安価で耐久性が高く、保存容量が無制限のオブジェクトストレージサービスです。オブジェクトストレージとは、データを固有のIDを持つ「オブジェクト」として扱う、ディレクトリのような階層構造を持たないストレージのことです。
AWSが提供する複数のストレージサービスのうち、S3は1TBあたりの保存料金が最も安く利用できます。
「オブジェクト」が保存される領域を「バケット」と呼びます。一つのバケットにはオブジェクトを無制限で保存できます。バケット名はグローバルで一意にする必要があり、作成したバケットのバケット名は変更できません。各オブジェクトにはリージョン、バケット名、オブジェクトキーが含まれたURLが付与され、HTTP/HTTPS経由でS3オブジェクトへアクセスできます。また、オブジェクトキーの先頭部分にプレフィックスと呼ばれる文字列を付与することで、フォルダのような階層構造でオブジェクトをグルーピングすることも可能です。

## 🆕 S3 Express One Zone の特徴（2023年11月 re:Invent で発表）

| 項目 | 内容 |
| :--- | :--- |
| **レイテンシ** | **一桁ミリ秒**（Standard比 最大10倍高速） |
| **バケットの種類** | 通常の「汎用バケット」ではなく **「ディレクトリバケット」** という新形式 |
| **データ配置** | **単一AZ内**の専用ハードウェアに保存（複数AZへの冗長化なし） |
| **リクエストコスト** | Standard比 **50%減**（超高スループット前提のため） |
| **コロケーション** | EC2/ECS/EKSと**同一AZに配置**することでさらに低レイテンシを実現 |
| **耐久性のトレードオフ** | 1AZのため、AZ障害時のデータ保護は他クラスより劣る |

> **【試験ポイント】**  
> - 「**一桁ミリ秒・超低レイテンシ**」が必要 → **S3 Express One Zone**  
> - 「**ディレクトリバケット**」というキーワードが出たら → S3 Express One Zone の話  
> - Standard の One Zone-IA（低頻度・1AZ）とは別物。Express One Zone は**超高頻度・超高速**が目的

# S3 オブジェクトロック機能

オブジェクトの**上書き・削除を防止**するためのWORM（Write Once Read Many）機能。  
バージョニングが有効なバケットでのみ使用できる。

## 2種類のロック方式

| 方式 | 保持期間 | 概要 |
| :--- | :---: | :--- |
| **リーガルホールド**<br>（Legal Hold） | **無期限** | 訴訟・調査などの法的保全が目的。`s3:PutObjectLegalHold` 権限を持つユーザーがON/OFFを自由に切り替えられる。 |
| **リテンションモード**<br>（Retention Mode） | **期限付き** | 保持期間を日数または日付で指定。期間中はモードごとのルールで保護される。 |

---

## リテンションモードの2サブタイプ

| 比較軸 | ガバナンスモード | コンプライアンスモード |
| :--- | :--- | :--- |
| **保護対象ユーザー** | `s3:BypassGovernanceRetention` 権限を**持たない**ユーザー | **ルートユーザーを含む全員** |
| **期間中の操作** | 権限なしユーザーは読み取り専用 | 全ユーザーが読み取り専用 |
| **モードの解除** | 権限を持つユーザーは解除・変更可 | 保持期間中は**誰も解除不可** |
| **保持期間の延長** | 権限を持つユーザーは変更可 | 延長のみ可（短縮は不可） |
| **ユースケース** | 誤操作防止・内部統制 | 金融・医療などの**法規制対応** |

## 全体構造まとめ

```
オブジェクトロック
├── リーガルホールド（無期限・手動ON/OFF）
└── リテンションモード（期限付き）
    ├── ガバナンスモード（特権ユーザーは解除可）
    └── コンプライアンスモード（誰も解除不可・最強）
```

> **【試験ポイント】**  
> - 「**誰も削除できない**（ルートユーザー含む）」→ **コンプライアンスモード**  
> - 「**管理者は変更できる**が一般ユーザーには制限」→ **ガバナンスモード**  
> - 「訴訟・法的保全のため**期間を定めずロック**したい」→ **リーガルホールド**  
> - コンプライアンスモードは保持期間の**短縮も解除も不可**（延長のみOK）
