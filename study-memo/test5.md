
- SQS
  - 標準キュー: 配信順序が考慮×
  - FIFOキュー: 配信順序が考慮○

  - SQSの状態はCloudWatchで監視できる
    - SQSのキュー状況を見てスケーリングさせる設計が可能


- SNS
  - 標準トピック: 配信順序が考慮×
  - FIFOトピック: 配信順序が考慮○

- SNS保護ポリシー
  - 「データ保護ポリシー」を設定することで、機密データなどをマスキングができる。
  - カスタムデータ識別子
    - 社内独自のデータなどを識別するためのもの
  - マネージドデータ識別子
    - クレジットカード番号やパスポート番号、メアドなどの一般的なパターンを識別する



- Amazon Kinesis Data Streams
  - アクセスログのリアルテイム分析
  - IoTセンサーデータの収集
  - 不正取引検知
  - ゲームのリアルタイムスコアリング
  - メトリクス監視

- データレイク
  - あらゆる形式のデータをそのまま大量に保存できる場所

- Lake Formation
  - 安全なデータレイクを構築・運用するためのマネージドサービス
  - 複数のデータソースからの取り込み、カタログ化、適切なアクセス権限などを一元管理
    - 名前付きリソースでのアクセスコントロール
    - タグ（LFタグ）ベースのアクセスコントロール

- DataSync
  - オンプレのストレージや他のクラウドからAWSへ大量データを高速に同期するサービス


- AssumeRole
  - 別のIAMロールの権限を一時的に借りる仕組み
  - STS（Security Token Service）の機能の一つ

各チームのアカウントで、S3バケットに対するアクセス権限を持つIAMロールを作成する。共有アカウントで、各チームで作成したIAMロールを引き受けるためのIAMロールを作成し、STS AssumeRole APIアクションを許可するポリシーを定義する。各チームのIAMロールに、共有アカウントのIAMロールを信頼するポリシーを定義する

S3
- S3レプリケーション
  他のS3バケットに自動的コピーする時にやる

- SageMaker Pipelines
  - 機械学習のワークフローを定義自動化管理するフルマネージドなCI/CDサービス

- リーガルホールド
  - 無期限
  - 手動で解除するまで継続
  - 用途
    - 調査完了までは削除しない

- ガバナンスモード（リテンション）
  - 権限を持たないユーザーに対して保持期間中は読み取り専用。
  - 権限ユーザーは更新もモード解除も可能
- コンプライアンスモード（リテンション）
  - 全ユーザーを指定した保持期間中は読み取り専用にする。
  - ルートユーザーを含めてモード解除は不可

インスタンスプロファイル
- EC2にIAMロールを紐づけるための入れ物
- EC2からS3やDynamoDBにアクセスしたいとき
  - IAMロールをEC2に付与する

- Compute Optimizer
  - 具体的にリソースの最適化提案をする
  - EX2インスタンス、Auto Scalingグループ、EBSボリューム、Fargate仕様のECSなど

- EBS
  - EC2の中のディスク（ストレージ）
  - ボリュームタイプ
    - 汎用SSD - gp3（コスパ重視・バランス型・ベストエフォート）
    - プロビジョンドIOPS - io2 - （ガチ性能要求）
    ※IOPS（1秒あたりのI/O）

- Cost Explorer
  - 過去のリソース使用量と費用を確認できる
  - 未来予測とかもできる
  - とにかく可視化・分析

- Trusted Advisor
  - ベストプラクティス診断（総合チェック）
  - 気づきをくれる

- IAMポリシー
  - 異なるアカウントからのアクセスポリシーはリソースに対して設定する
  - そもそもIAMポリシーは
    - 誰が、どのリソースに、どの操作を許可するか


- Step Function
  - 複数のAWSサービスをワークフローとして連携して処理を自動化するフルマネージド型のサービス
    - サーバーレス
    - 専用のGUI
    - 人間の手動操作待機など、複雑なワークフローを定義できる
    - オンプレ環境や外部サービスとも連携可能
