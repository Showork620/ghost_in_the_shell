# Amazon VPC (Virtual Private Cloud) の概要

Amazon VPCは、AWSアカウント専用の仮想ネットワークを定義できるサービスです。AWSクラウド内の他の仮想ネットワークから論理的に切り離されており、IPアドレス範囲、サブネットの作成、ルートテーブルやネットワークゲートウェイの設定などを完全に制御できます。

VPCは1つのリージョン内に複数作成できます。そして、1つのVPCはリージョン内にある全てのAZにまたがります。VPCで作成されるネットワーク空間は、各VPCが独立しているプライベートネットワークです。そのため、デフォルトでは他のVPCやインターネットと通信できません。

## 主要な構成要素
- **サブネット**: VPC内のIPアドレス範囲のセグメント。インターネットに直接アクセス可能な「パブリックサブネット」と、直接アクセスできない「プライベートサブネット」に分けられます。
- **ルートテーブル**: サブネット内のトラフィックがどこに向かうかを制御するルールのセットです。
- **インターネットゲートウェイ (IGW)**: VPCとインターネット間の通信を可能にするコンポーネントです。
- **NAT ゲートウェイ**: プライベートサブネットにあるインスタンスがインターネットにアクセスできるようにし、外部からの不要な接続をブロックします。
- **セキュリティグループ**: インスタンス単位の仮想ファイアウォール（ステートフル：戻りトラフィックを自動許可）。
- **ネットワークACL (NACL)**: サブネット単位の仮想ファイアウォール（ステートレス：戻りトラフィックも明示的に許可が必要）。

---

 ## Elastic IPアドレス

 - インターネットと通信可能な固定のパブリックIPアドレス。AWSリソースを終了しても継続され、料金も発生。同一のIPアドレスを別のリソースに割り当て可能。

 ## インターネットゲートウェイ

 - VPC内のAWSリソースとインターネットを接続させるためのもの。
 - インターネットゲートウェイへのルーティングを設定したサブネットはパブリックサブネットとなる


 ## Network Firewall

 AWS Network Firewall は、VPC 全体に対する高度なネットワーク保護（ステートフルなパケットインスペクション、侵入検知・防止など）を提供するマネージドサービスです。

 ## ネットワークACL (NACL)

 ネットワークACL（Network Access Control List）は、サブネット単位で適用される仮想ファイアウォールです。

 - **サブネットレベルの保護**
    - 1つまたは複数のサブネット全体の境界で、出入りするトラフィックをフィルタリングします。
 - **ステートレス (Stateless)**
    - 「行き」の通信を許可しても、「帰り」の通信は自動的には許可されません。インバウンドとアウトバウンドの両方で、ルールを明示的に設定する必要があります。
 - **許可 (Allow) と 拒否 (Deny) の両方を指定可能**
    - 特定のIPアドレスからの攻撃を遮断したい場合など、明示的に「拒否」ルールを作成できます（セキュリティグループは「許可」のみ設定可能です）。
 - **番号順の評価**
    - 各ルールには番号があり、番号の小さい順に評価され、最初に一致した結果（許可または拒否）が適用されます。

| 特徴 | セキュリティグループ | ネットワークACL |
| :--- | :---: | :---: |
| 適用単位 | インスタンス単位<br>（EC2等） | サブネット単位 |
| 状態管理 | ステートフル<br>（戻りは自動許可） | ステートレス<br>（往復とも許可が必要） |
| ルール内容 | 許可のみ | 許可と拒否の両方 |
| 評価順序 | 全てのルールを並列評価 | 番号順に評価し、最初に一致したものを適用 |

## ルートテーブル

VPCのルートテーブルは、VPC内での通信において、どのネットワークへデータを転送するかを定義する機能です。VPC内の各サブネットは、1つのルートテーブルを関連付けることができます。ルートテーブルが関連付けられていないサブネットは、VPC全体に適用される「メインルートテーブル」に従ってルーティングが行われます。

 - VPC内の各サブネットは、1つのルートテーブルを関連付けることができます。
 - ルートテーブルにインターネットゲートウェイをルーティングを設定することで、パブリックサブネットとなり、インターネットに接続可能になる。
 - ルートテーブルの紐付けがないサブネットは、VPC全体に適用される「メインルートテーブル」に従ってルーティングする。

 --

 ## Egress-Onlyインターネットゲートウェイ

  - VPCからインターネットへ（Egress）の接続開始要求は通しますが、インターネットからVPCへ（Ingress）の接続開始要求は通しません。
  - Egress-Onlyインターネットゲートウェイを利用するには、VPCにEgress-Onlyインターネットゲートウェイを作成し、プライベートサブネットのルートテーブルに、送信先が「::/0（デフォルトルート）」、ターゲットが「Egress-OnlyインターネットゲートウェイID」となるルートを追加します。
