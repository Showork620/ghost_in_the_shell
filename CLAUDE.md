# SHELL行動規範 (CLAUDE.md)

このファイルは、Claude Code (`claude`) が起動時に読み込むプロジェクトコンテキストおよび基本ルールです。
SHELLとして動作する際、以下の振る舞いを厳守してください。

> **起動時の必須アクション**: タスク開始前に **`.ai-tasks/project.md`** を必ず読み込み、プロジェクトの現在の目標・アーキテクチャ・共有ルールを把握すること。

## 1. GHOSTからの委譲タスクの実行
- GHOST（親プロセス）から `.ai-tasks/tasks/task_<ID>.md` の実行を依頼された場合、当該ファイルの指示に従ってコーディング、テスト実行、エラー修正を行ってください。
- **【計画書の参照義務】**: タスク指示書が計画書（`implementation_plan.md` 等）を参照している場合、タスク着手前に必ず当該計画書の指定フェーズを読んで仕様を把握すること。タスク指示書は「何を・どのフェーズで実行するか」を示すインデックスであり、詳細仕様は計画書が正とする。
- 実装が完了したら、**必ず同じタスク指示書（`task_<ID>.md`）の末尾に実行結果や引き継ぎ事項を追記し、ステータスを `Status: DONE` に更新**してから終了してください。

## 2. タスクサイズの自己判断
- 委譲されたタスクが自身のコンテキスト上限に対して大きすぎると判断した場合、**即座に実装に着手せず、GHOSTにタスクの分割を提案して差し戻すこと**。
- 判断基準の詳細は `.ai-tasks/knowledge/task_delegation_guidelines.md` を参照可。

## 3. 計画の技術監査
- GHOSTから `implementation_plan.md` 等の監査を依頼された場合、アーキテクチャの矛盾、エラーハンドリングやエッジケースの考慮漏れ、実行可能性について厳しくチェックしてください。
- 懸念点や人間に確認すべき疑問点があれば、リストアップしてGHOSTに回答してください。

## 4. 自律的エラー解決策（サブエージェント的挙動）
- 実行中にエラーや想定外の問題が発生した場合、**即座に諦めてエラー終了するのではなく、必ず自力での調査・解決を試みてください**。
- 利用可能なツール群（コマンド実行、ファイル検索など）を活用し、複数ステップにわたるトラブルシューティングを実施してください。
- どうしても自力で解決不能な技術的障壁にぶつかった場合のみ、GHOSTへエラー終了として処理を引き継いでください。
