# AWS分析カテゴリの詳細化とサービス間連携の統合計画

ユーザーから提供された情報を元に、既存の `AWSSAA_Dictionary_2026.md` の各サービス項目を詳細化します。全体俯瞰セクションは設けず、各カード（サービス項目）内で「サービス間の繋がり」を具体的に解説する方針とします。

## ユーザーレビューが必要な事項

> [!IMPORTANT]
> **サービス間の連携（データの流れ）の記述場所**
> 各サービスの「SAA試験でのポイント」および「どういう時に使うサービスか」のセクションに、前後の収集・保存・分析サービスとの具体的な連携フローを追記します。
> 例：Firehose の項目に「S3 への自動配送・データレイク構築の起点」等の記述を追加。

## 提案する変更内容

### 1. 既存サービスの記述詳細化 [MODIFY]
以下のサービス項目を、提供された「判断軸」や「役割」を反映して更新します。
- **Amazon S3**: 役割に「データレイク（分析の起点）」、ポイントに「Parquet 変換による最適化」を追記。
- **Amazon Athena**: 役割に「ad-hoc / 軽量分析」、ポイントに「S3 上のログを直接 SQL 分析」「サーバーレス設計」を追記。
- **Amazon Redshift**: 役割に「大規模 / BI / 高速集計」、ポイントに「JOIN が多い複雑なクエリ」「BI 用途」を追記。
- **AWS Glue**: 役割に「サーバーレス ETL」、ポイントに「JSON から Parquet への変換」「データカタログ作成」を追記。
- **Amazon EMR**: 役割に「分散処理（Spark/Hadoop）」、ポイントに「カスタム処理」「超大規模」を追記。
- **AWS Lake Formation**: 役割に「権限管理（テーブル/列レベル）」、ポイントに「データ統制」を追記。
- **Amazon Kinesis Data Streams**: 役割に「リアルタイム処理前提」、ポイントに「独自リアルタイム処理との連携」を追記。
