# Codexへの依頼プロンプト：AWS SAA 2026年版 統合辞書の作成

あなたはAWS認定ソリューションアーキテクト - アソシエイト（SAA）試験の学習アシスタントです。
以下の要件に従い、プロジェクト内の散在している学習メモを統合して1つの「SAA用辞書リスト」を作成してください。

## 1. 入力ソース（読み込み・抽出対象）
ワークスペース内の以下のファイル群を解析し、言及されているAWSサービスや重要キーワードをすべて抽出してください。
*   `study-memo/WellArchitectedFramework_サービス詳細.md`
*   `study-memo/DR設計_読み上げ原稿.md`
*   `study-memo/test1.md` 〜 `study-memo/test6.md` 等

## 2. 出力・構成フォーマットの要件
*   **ソート順**: A-Z順、その後に五十音順（その他の用語）とするフラット構成。
    *   **重要**: サービス名先頭の「Amazon」や「AWS」は無視して、実質的なシステム名称でソートすること。（例: `Amazon EC2` は `E` から始まるグループに配置）
*   **各項目の記載要素**: 各サービスに対し、以下の5項目を過不足なく出力すること。
    1.  **概要**: 簡潔な説明（1〜2行程度）
    2.  **カテゴリ**: 機能カテゴリ（明記する：コンピューティング、ストレージ、ネットワーク等）
    3.  **タグ**: `#パフォーマンス効率` `#コスト最適化` `#DR設計` など検索・絞り込み用タグ
    4.  **SAA試験でのポイント**: 2026年のアップデート・出題傾向を反映した試験対策上の重要点
    5.  **どういう時に使うサービスか**: 具体的なユースケース（Webサーバ、ログ保存先、メッセージングなど）

## 3. 記載例（出力フォーマット）
```markdown
### Amazon EC2 (Elastic Compute Cloud)
- **カテゴリ**: コンピューティング
- **タグ**: `#パフォーマンス効率` `#コスト最適化` `#DR設計（ウォームスタンバイ・パイロットライト）`
- **概要**: AWS上の仮想サーバー。マルチAZ配置で冗長化が可能。
- **SAA試験でのポイント**: マルチAZ構成とAuto Scalingの組み合わせによる信頼性確保とコスト最適化が頻出。
- **どういう時に使うサービスか**: Webサーバ、アプリケーションサーバ、バッチ処理サーバなど
```
