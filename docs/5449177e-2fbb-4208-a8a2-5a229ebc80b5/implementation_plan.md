# [計画] カテゴリの再編と不足サービスの調査依頼 (GHOST-SHELL 連携)

現在の統合辞書 (`AWSSAA_Dictionary_2026.md`) の整理をさらに進めるため、SHELL (Claude CLI) に対し、カテゴリの親子構造化（階層化）と不足サービスの網羅的調査を依頼します。

## ユーザーレビューが必要な事項

> [!IMPORTANT]
> **カテゴリ階層化の方針**
> 現在の「セキュリティ/XXX」と同様に、他の主要カテゴリ（ネットワーク、データベース、ストレージ等）も「親/子」形式で再編することを提案します。
> 例: `ネットワーク/接続系`、`データベース/リレーショナル` 等。

## 提案する変更内容

### 1. 調査タスクの作成と SHELL への委譲 [NEW]
- **タスクファイル作成**: `.ai-tasks/tasks/task_audit_dictionary_2026.md` を作成。
- **調査項目**:
    - 全 100+ インスタンスのカテゴリを抽出し、親カテゴリ（Networking, Storage, Database, Analytics, AppIntegration 等）を定義。
    - SAA-C03 (2026版) の試験範囲に含まれるが、現在の辞書に不足しているサービス（例: App Runner, SageMaker, AI/ML 系等）をリストアップ。
    - 既存の「セキュリティ」サブカテゴリとの整合性を確認。
- **SHELL 起動**: `claude` コマンドを使用（`--permission-mode acceptEdits`）。

### 2. 調査結果の集約 [NEW]
- `study-memo/investigation_report_categories.md` に調査結果（階層化案と不足サービスリスト）を出力。

### 3. 次のステップの提案
- 調査結果に基づき、以下の選択肢をユーザーに提示します。
    - A: 提案された階層構造に一括変換
    - B: 不足サービスの追加のみ実施
    - C: 特定のカテゴリ（例：ネットワークのみ）をまず詳細化
