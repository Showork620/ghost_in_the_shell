# モバイル表示最適化（縦・横スクロール修正）完了報告

モバイルブラウザにおけるビューポートHeight/Widthの扱いを最適化し、スクロール体験とレイアウトの安定性を向上させました。

## 実施内容

### 1. ビューポート高さ（Height）の最適化
- `body`, `#root`, `.overlay-panel` 等の主要なコンポーネントに `dvh` (Dynamic Viewport Height) を適用しました。
- これにより、モバイルブラウザ（iOS Safari等）でアドレスバーが表示・非表示される際の、オーバーレイの底が隠れたりスクロールが途切れたりする問題を解消しました。
- 古いブラウザ向けに `100vh` のフォールバックを併記しています。

### 2. 横揺れ・横スクロールの防止
- 画面幅指定から `vw` ユニットを排除し、`%` ベースまたは `calc(100% - ...)` に変更しました。
- スクロールバーの幅が `100vw` に含まれることで発生していた不要な横スクロール（横揺れ）を防止します。
- `body` に `overflow-x: hidden` を追加し、予期せぬ要素のハミ出しによる揺れを遮断する安全策を講じました。

### 3. トップバー・オーバーレイの修正
- `.topbar-wrap` の `vw` 依存マージンを廃止し、安全な負のマージン指定に変更。
- `@media (max-width: 640px)` 内の各要素のサイズ指定を、最新の最適化ルールで再定義。

## 検証のお願い

> [!IMPORTANT]
> GHOSTの行動規範により、ブラウザ上での動作確認はユーザー側で実施いただく必要があります。

実機またはブラウザのインスペクタ（モバイルエミュレーション）にて、以下の点をご確認ください。
1. **縦スクロール**: オーバーレイを表示した際、アドレスバーが伸縮しても最後まで（ガイドカードの末尾まで）正しくスクロールできるか。
2. **横揺れ**: 左右にスワイプしたときに、画面全体がガタついたり、右側に不要な隙間が発生したりしないか。
3. **操作性**: 各オーバーレイ（Hero, 絞り込み, 比較シート）の開閉が正しく動作するか。

問題なければ、このセッションを終了し Git コミットを推奨します。
