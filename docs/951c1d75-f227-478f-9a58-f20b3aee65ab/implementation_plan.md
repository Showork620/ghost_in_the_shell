# モバイル表示におけるビューポートHeight/Widthの最適化計画 (Audit Revised)

SHELL による監査結果に基づき、以下の点を強化した修正計画を実施します。特にメディアクエリ内の指定漏れ防止と、`vh` ユニットのフォールバックを徹底します。

## ユーザーレビュー要求事項
- **フォールバック設計**: `dvh` ユニットをサポートしない古いブラウザ向けに、必ず `vh` を併記して定義します。
- **topbar-wrap の挙動**: スクロールバーによる横揺れを完全に防ぐため、`vw` に依存しない全幅化手法（または `calc(100% + ...)` への変更）を検討します。

## 提案される変更点

### CSS 最補正

#### [MODIFY] [styles.css](file:///Users/rhyolite/practice/awssaa_in_the_ghost/src/styles.css)

- **縦方向の高さ修正 (フォールバック付)**:
    - `body`, `#root`, `.overlay-panel` 等の高さ指定箇所に `height: 100vh; height: 100dvh;` の形式を適用。
    - **重要**: `@media (max-width: 640px)` 内の `.overlay-panel` (`height: calc(100vh - 12px)`) も `dvh` に置換。
- **横方向の幅修正 (横スクロール防止)**:
    - **.topbar-wrap**: `margin: -24px calc(50% - 50vw) 18px` を廃止し、親の `app-shell` の幅指定に依存しない、または `width: 100%;` 等で安全に収まる形に修正。
    - **.topbar**: `width: min(1180px, calc(100vw - 24px))` を `calc(100% - 24px)` または `100%` ベースの計算に変更。
    - **.overlay-panel**: `width: calc(100vw - 12px)` を `100%` または `calc(100% - 12px)` に変更し、絶対的な `vw` による溢れを完全に排除。
- **cards-area の余白**: `padding-bottom: calc(100dvh + 24px)` への修正。

## 検証計画
